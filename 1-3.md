# Javascript에서 var보다 let, const 사용을 지향해야하는 이유를 설명해보세요.

> 자바스크립트는 es6문법 도입전까지 var키워드만 사용해 변수를 선언.
> 그로인해 변수의 재선언, 재할당이 가능하여 유연한 변수 선언 면에서는 장점이지만,
> 코드양이 많아질수록 유지관리가 힘들고, 에러를 찾기도 힘들어짐.

 ### hoisting - 인터프리터가 변수와 함수의 메모리공간 선언 전에 미리 할당하는 것. (순서대로 진행하니까

> var로 선언한 변수는 호이스팅을 한다면 undefined로 변수를 초기화.
> 코드 error가 아니라 undefined가 찍힌다면…잘못을 인지하지 못하고 작성하다 큰 문제가 생길 수 있음
>let과 const가 선언되기 이전에 선언되지않은 변수를 사용하면 reference에러로 not defined라고 떠서 좀 더 예상치 못한 에러를 빠르게 발견할 수 있다.
> let,const로 선언한 변수는 호이스팅 시에 변수를 초기화하지 않음.    

### 변수의 유효범위
> var = function scope : 함수 내부에 선언된 변수만 지역변수로 간주
> let, const = block scope : 함수 뿐만 아니라 for,if문에서도 지역변수로 간주함. << ??

# Javascript에서 함수 선언식과 함수 표현식에 대해 설명해보세요. 

함수 선언식?      
* 함수명 지정, 별도의 할당명령이 없는 것 ( function sum(a,b) )    
함수 표현식?       
 * 정의한 함수를 별도의 변수에 할당하는 것 (const sum = function(a,b))      

### 둘의 차이는 호이스팅에서 발생
> 함수 선언식 - 함수 전체를 호이스팅, 정의된 범위 맨 위로 호이스팅되어 함수 선언 전 함수를 사용가능.    
> 함수 표현식 - 별도의 변수에 할당하게 되는데, 변수의 선언부와 할당부를 나누어, 선언부만 호이스팅 함.    
> 함수 선언식으로 작성한 함수를 전역적으로 선언하게된다면 중복적으로 동명의 함수를 쓰게될 때 ….. No….   
> 이에 함수표현식으로 작성하면 방지할 수 있음!

###  Element ; react에서 UI를 구성하는 가장 작은 단위
> 1. Type : React element의 유형을 나타내는 문자열 또는 함수
> 2. Props : React element에 적용될 속성 및 값의 집합

> JSX : JavaScriptXML의 약자

> JSX로 작성된 코드는 앞서 설명한 babel과 같은 트랜스파일러를 통해 JavaScript코드로 변환됨

# 브라우저의 DOM과 React의 virtual DOM에 대해서 설명해보세요. 

### 브라우저 DOM
		>DOM  ; Document Object Model의 약자
		> 브라우저에서 웹 페이지 내에서 요소를 선택하고 조작할 수 있는 API와 함께 사용
		> JavaScript코드를 통해 DOM을 제어하여 동적으로 내용이나 기능을 추가할 수 있음.
		> 가상 돔에 비해 리플로우 및 리페인팅 발생 많음
		> 속도적 부분 및 많은 일을 수행하다 버그 발생 및 브라우저가 죽는 일 등을 개선시켜야 해서 가상돔이 나오게되는 배경이 됨
			
			
### React - Virtual DOM
		* 리액트에서 사용되는 개념으로, 브라우저의 돔과 유사한 구조를 가지고 있으나,
			메모리 상에 존재하는 가상 돔.
		* 데이터 변경 시마다 이전 가상돔과 현재 데이터를 비교하여 변경된 부분만 실제 돔에 반영하여 성능을 최적화 시킴.
		* 위와 같은 방식으로 불필요한 화면 업데이트를 줄이고, 앱의 성능을 향상
		* react.createElement() 함수로 생성된 요소로 이루어져 있음.
		* 각 요소는 태그명, 속성값, 자식요소 등의 정보 포함.
		* 실제 환경에서 발생할 수 있는 오버헤드 및 복잡도 문제를 해결하기위한 방법.
